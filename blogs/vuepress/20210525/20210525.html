    基于Vuepress搭建个人博客

    Joe Lang 5/25/2021 Vuepress

    如你所见，本博客就是基于 Vuepress 搭建的个人博客。

    # 关于 Vuepress

    其实 Vuepress 不仅限于搭建个人博客，还可以是个人网站以及任何静态网页。当然 Vuepress 的好处还是在于它是基于 Markdown 的，对于使用 Markdown 写作的用户来说非常方便。

    Vuepress v1 版本基于 Vue 2，而 VuePress v2 基于 Vue 3，且支持 TypeScript。

    考虑到目前 v1 版本的主题和插件比较丰富，因此本博客是基于 Vuepress v1 的。之后将会写基于 v2 版本的开发指导。

    # 基于主题创建 Vuepress 项目

    基于已有的主题创建项目是构建博客最快的一种方式，所以优先介绍这种方式。

    市面上有一些做的比较漂亮的 Vuepress 主题，可以利用现有的这些主题快速搭建博客。然后只需要进行一些个性化的设置以及上传自己的 md 格式的内容即可。下面介绍几个我认为做的比较优秀的主题。

    # 官方默认主题

    默认主题的样式请参考官方文档 (opens new window)。

    npm install -g yarn

    # 使用脚手架安装 Blog 主题（更推荐）

    使用官方提供的脚手架能一键安装。

    create 命令已经废弃了，由 create vuepress-site 替代。但是 create vuepress-site 不提供 Blog 主题的安装，只有文档主题。所以这里还是用 create。

    # 遇到 Select the boilerplate type 选择blog

    # reco 主题

    Demo：https://vuepress-theme-reco.recoluan.com/

    Github：https://github.com/vuepress-reco/vuepress-theme-reco

    实现了自适应，深色模式，有多个插件可用。

    有两种安装 reco 主题的方式。

    # 使用主题脚手架安装（更推荐）

    该主题提供了脚手架，可以一键安装：

    yarn global add @vuepress-reco/theme-cli
#在空项目目录下
theme-cli init
yarn install

    2）基于默认主题安装 reco

    所以可以从零开始或基于默认主题构建 Vuepress 项目，再引入 reco 主题。

    yarn add vuepress-theme-reco

    theme: "reco",

    如果嫌麻烦，还是用主题脚手架安装吧。

    # 创建脚本 deploy.sh

    因为每次更新博客内容都要 push 一遍，所以干脆写一个脚本，方便一键上传。

    在项目根目录下新建 deploy.sh

    编辑 deploy.sh

    #打包项目
npm run build

#打包后生成dist目录
cd .vuepress/dist

#git初始化
git init
git add -A
git commit -m 'deploy'
#这里更改成自己的github地址
git remote add origin git@github.com:JoeLang2017/newsletter.git
git pull --rebase origin master
git push --force origin master

#删除dist目录
cd ../
rm -rf dist

    # 可能出现的问题

    在将代码 push 到远程仓库时，可能会遇到ssh: connect to host github.com port 22: Connection timed out的错误。解决办法有以下几种：

    1）可能是由于 22 端口被占用，试着重启一下；

    2）可能是由于 SSH 公钥丢失。如果能找到C:\Users\xxx\.ssh目录，则SSH公钥没有丢失。否则需要重新生成公钥：

    ssh-keygen -t rsa -C "xxx@xx.com"
    1

    生成的公钥在C:\Users\xxx\.ssh目录下的id_rsa.pub文件。

    然后打开 github ，在settings下，SSH and GPG keys下new SSH key，然后将id_rsa.pub里的内容复制到Key中，完成后Add SSH Key。

    再次执行ssh -T git@github.com看能不能连上。

    3）重新配置端口为 443，在存放公钥私钥(id_rsa 和 id_rsa.pub)的文件里，新建config文件，内容如下：

    Host github.com
User YourEmail@163.com
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443
